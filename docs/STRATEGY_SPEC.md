# Strategy Specification (AST)

## Purpose
The Strategy object is the Abstract Syntax Tree (AST) of C³. It is the intermediate, semantic representation of *how* a problem should be solved, generated by the Problem Analyzer/Planner (the front-end) and consumed by the Compiler (the middle-end) to emit RIR.

## Concept
Humans can author Strategy objects. 
The Compiler emits RIR. 
The Runtime executes RIR. 
**The Runtime never executes Strategy directly.**

## Schema

```yaml
version: "1.0"
problem_class: "analytical_reasoning"

execution_profile:
  confidence_target: 0.95
  parallelism_allowed: true
  fail_fast: false

# Identifies the requisite classes of Knowledge Primitives
knowledge_requirements:
  - "empirical_data"
  - "historical_records"

# Identifies the requisite classes of Reasoning/Execution Primitives
reasoning_requirements:
  - "deduction"
  - "summarization"

# Identifies the requisite Verification Primitives
verification_requirements: "required"
```

## Compilation Contract
When the Compiler receives a Strategy object, it reads the requirements and maps them to concrete Primitive ISA opcodes:
- `knowledge_requirements` -> `KNOW.*` opcodes (e.g., `KNOW.RETRIEVE`)
- `reasoning_requirements` -> `REAS.*` or `EXEC.*` opcodes (e.g., `REAS.INFER`, `EXEC.PYTHON`)
- `verification_requirements` -> `VERI.*` opcodes (e.g., `VERI.VERIFY`)
